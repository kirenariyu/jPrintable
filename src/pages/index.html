Print Responsively

Please take note that you don't need jQuery, Foundation and LESS or DataTables to use this plugin. I only used them here for the design. The design of your web's all yours. You'll be only needing are the following:

  1. print-responsively.css for @media print
  2. print-responsively.js for config

The cons for this plugin is you need to place the size of the paper and will only be responsive for that size of the paper.

Introduction

Print Responsively aims to render webpage into a specific paper simply, properly and responsively in every browser; eliminating the default footer and header of user's web browser.

Before, there were rules in CSS that we can use for @media print so we could print our desired webpages properly. But, unfortunately, some of them were already deprecated. So I made this plugin to help others give idea or use it (that would be an honor) for their website or web application.

Getting Started

Print Responsively is made purely in JavaScript.

I created a simple website in this repository to give you some examples on how this plugin works. But you will be only needing these files to add in your document's head:

  • css/print-responsively.css
  • js/print-responsively.js

Then add this code to your code:

$(document).printResponsively(options);

Cheat Sheet (CSS)

You may add additional rules depending on the set design. Just take note that sometimes float elements can destroy your design.

If you encountered some display problem, you also need to look carefully the parent element/s because they can affect the look of your page on print and this can really waste a lot of your time. Your partner-in-crime is inspect console in browsers and try experimenting in styles.

Class Description Prerequisite
.hide-print This will hide the visible element on print.
.show-print This will show the hidden element on print.
.header-area Place this class in the parent container of the header where its associated contents are also included. You may combine this class with .footer-area class if there's also a footer incorporated. header
.footer-area Place this class in the parent container of the footer where its associated contents are also included. You may combine this class with .header-area class if there's also a header incorporated. footer

Cheat Sheet (Config)

Option Description Default
buttonClass Allows you to add more classnames to the print button. For multiple class, add space as delimiter just like inputting classes in HTML. null
buttonContainer A required option if you want to place a print button in your page. Pass it with node name or id (#id) of the element or class (.class) of the element/s if you like to put the print button in multiple places of your page. null
buttonText Allows you to change the default text of the print button. You may also add the size name by adding <size> in the string. This option is recommended if you have multiple page sizes. 'Print'
container Assigns a container element to be the only content to be shown on print. For multiple containers, add comma as delimiter. Please be sure to include selector. 'body'
cssContainer Specifies the container where the dynamically added css file will be placed. The default would be in the first head of the document. If there's no cssPath but you placed this option, it will occur an error. 'head'
cssPath This is used for dynamically adding of the css file of Print Responsively by passing its file or url path. null
footer Assigns a container element to be the footer of the page on print. There should be one for each .footer-area. null
header Assigns a container element to be the header of the page on print. There should be one for each .header-area. null
keypress If true, it will allow you to print using ctrl and P with the configured layout. false
margin Specifies the margin for all pages. Pass margin the same way in css. There are still no function where you can change margin on multiple pages. '0'
orientation Allows you to change the layout of the pages. There are only two choices for this option: portrait and landscape. 'portrait'
size Allows you to change the paper size and add more wih delimiter comma. If the desired size is not in the list, you may add manual size e.g. <120cmX130cm> in widthXheight format. The measurements should be in lowercase and the delimiter 'X' should be in uppercase. 'letter'

Styling in @media print

All margin and padding from the body or the selected container will reflect in the print. There are some instances that @media print is so painful to work on for not all styles can be read by browser's print or page setup.

You may search from Google or ask for help in StackOverflow regarding some issues of your media print. But some of them have been asked and answered

Configuration

buttonClass

Allows you to add more classnames to the print button. For multiple class, add space as delimiter just like inputting classes in HTML.

Each buttons' first classname is .print-button. If you want to add default style to print button, you may call .print-button to your css file.

Please take note that the buttonContainer option is required to turn on the automatically created print buttons.

printResponsively({
    buttonContainer: '#buttonContainer',
    buttonClass: 'class-1 class-2'
});

buttonContainer

This option automatically adds print button in a specific container. Just call the id or classname of the desired container.

If there's no buttonContainer configured in Print Responsively, then there'll be no print button in the page. Otherwise, you may always turn on the keypress to enable Ctrl + P shortcut key to print with your configured Print Responsively layout.

printResponsively({
    buttonContainer: '.buttonContainers-1, .buttonContainers-2, #sub-buttonContainer'
});

buttonText

This allows you to change the default text which is 'Print' in print button.

If there are multiple sizes, there would be multiple buttons so I would recommend to add <size> for it will be replaced with the paper size of each specific print button.

Please take note that the buttonContainer option is required to turn on the automatically created print buttons.

printResponsively({
    buttonContainer: '#buttonContainer',
    buttonText: 'Print Me (<size>)'
});

container

This allows you to specify contents to be shown in page print.

Instead of using the class .show-print when there's only one content to be shown, I recommend to use this option and add the id or classname of the element as value.

printResponsively({
    container: '.containers-1, .containers-2, #someSpecificContainer'
});

cssContainer

This option allows you to add the css file of Print Responsively in a specific element or elements. Otherwise, its default value is in the (first, if multiple) head element.

To be able to use this option, cssPath must also be called.

printResponsively({
    cssContainer: '.i-want-to-put-it-on-a-div'
    cssPath: '//url-path-to-the-print-responsively-css-file'
});

cssPath

This allows you to dynamically add the css file.

printResponsively({
    cssPath: '//url-path-to-the-print-responsively-css-file'
});

footer

This lets you add footer in page setup. Just return the id of the element or classname as long as its the only element with classname in the page.

Don't forget to add class .footer-area in footer's parent.

The contents of the footer should be also inside footer's parent.

printResponsively({
    footer: '#footer'
});

header

This lets you add header in page setup. Just return the id of the element or classname as long as its the only element with classname in the page.

Don't forget to add class .header-area in header's parent.

The contents of the header should be also inside header's parent.

printResponsively({
    header: '#header'
});

keypress

The fefault value for this option is false. If true, this enables the shortcut key Ctrl + P to print with the configured layout in Print Responsively.

printResponsively({
    keypress: true
});

margin

To be able to add margin each page, use this option and set value of the desired margin. Else, the default value is 0.

The value for the margin has the same accepted format and value in css.

printResponsively({
    margin: '20px 15px'
});

orientation

This allows you to change the orientation of your page to landscape or portrait.

The default value is portrait.

printResponsively({
    orientation: 'landscape'
});

size

This allows you to change the default paper size of your page. You may add multiple sizes with comma (,) delimiter but it will also add multiple print button.

If there are no sizes in the list, you may add it in widthXheight, delimiting with capital X, e.g. 120mmX210mm.

The default value is letter size.

printResponsively({
    size: 'A4'
});
